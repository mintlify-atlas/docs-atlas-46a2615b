This guide will walk you through your first search and label printing operation with Daikin Etiquetas.

<Info>
Before starting, ensure you have [installed Daikin Etiquetas](/installation) and have access to supplier part numbers you need to look up.
</Info>

## Launch the Application

<Steps>
  <Step title="Open Daikin Etiquetas">
    Double-click the `Etiquetas.exe` file or the desktop shortcut.
    
    The application window will open, displaying:
    - The Daikin logo at the top
    - A search input field labeled "Supplier Part Number"
    - A Search button
    
    <Note>
    The application uses a custom window frame with minimize and close buttons in the top-right corner. You can drag the top bar to move the window.
    </Note>
  </Step>
</Steps>

## Perform Your First Search

<Steps>
  <Step title="Enter Supplier Part Number">
    Click on the text field that displays "Supplier Part Number" as placeholder text.
    
    The placeholder text will clear automatically, and you can type the supplier part number you want to look up.
    
    ```text
    Example: ABC123456
    ```
    
    <Info>
    The input field accepts alphanumeric characters. The text will appear in bold white font once you start typing.
    </Info>
  </Step>
  
  <Step title="Click Search">
    Click the **Search** button to query the Warehouse Support Dashboard API.
    
    The application will:
    1. Validate that you've entered a part number
    2. Send an HTTP request to the API with your search query
    3. Display results in a table below the search field
    
    <Warning>
    If you click Search without entering a part number, you'll see a warning message: "Please enter a Supplier Part Number"
    </Warning>
  </Step>
  
  <Step title="View Search Results">
    After a successful search, a results table will appear with three columns:
    
    | Column | Description |
    |--------|-------------|
    | **Print** | Button to print label for this part |
    | **Supplier Part Number** | The supplier part number you searched for |
    | **Daikin Part Number** | The corresponding Daikin part number |
    
    The results table will show all matching Daikin part numbers for the supplier part number you entered.
    
    <Info>
    If no results are found, the table will be empty. Verify the supplier part number is correct or try a different part number.
    </Info>
  </Step>
</Steps>

## Print a Label

<Steps>
  <Step title="Select Part to Print">
    In the results table, locate the row with the Daikin part number you want to print.
    
    Each row has a **Print** button in the first column.
  </Step>
  
  <Step title="Click Print Button">
    Click the **Print** button for the desired part number.
    
    A confirmation dialog will appear showing:
    ```
    Imprimiendo etiqueta para:
    Supplier PN: [Your Supplier Part Number]
    Daikin PN: [Corresponding Daikin Part Number]
    ```
    
    Click **OK** to proceed with printing.
    
    <Note>
    The current implementation shows a confirmation dialog. In production environments with printer integration, this will send the label directly to your configured label printer.
    </Note>
  </Step>
  
  <Step title="Verify Label">
    Once printed, verify that the label contains:
    - Correct supplier part number
    - Correct Daikin part number
    - Any additional formatting required for your warehouse
  </Step>
</Steps>

## Understanding the Search Process

When you click Search, the application performs the following operations:

```csharp
// 1. Validates input
if (string.IsNullOrWhiteSpace(txtSupplierP_N.Text) || 
    txtSupplierP_N.Text == "Supplier Part Number")
{
    MessageBox.Show("Please enter a Supplier Part Number");
    return;
}

// 2. Constructs API request
string url = $"https://azuappsrvuat01v.mcquay.com/SupportDashboard/Home/GetReportData?id=2167&parameters={{\"supplier_part_number\":\"{supplierPartNumber}\"}}";

// 3. Sends HTTP GET request
HttpResponseMessage response = await client.GetAsync(url);

// 4. Processes JSON response
var results = JsonSerializer.Deserialize<List<Dictionary<string, object>>>(jsonResponse);

// 5. Populates results table
foreach (var result in results)
{
    daikinPartN_tbl.Rows.Add(
        "Print",
        result["Supplier_Part_number"],
        result["Daikin_Part_Number"]
    );
}
```

## API Response Format

The Warehouse Support Dashboard API returns data in the following JSON format:

```json
[
  {
    "Supplier_Part_number": "ABC123456",
    "Daikin_Part_Number": "DKN789012"
  },
  {
    "Supplier_Part_number": "ABC123456",
    "Daikin_Part_Number": "DKN789013"
  }
]
```

<Info>
A single supplier part number may map to multiple Daikin part numbers. The application displays all matches in the results table.
</Info>

## Common Workflows

### Single Part Lookup

For occasional lookups:

1. Launch application
2. Enter supplier part number
3. Click Search
4. Print label(s) as needed
5. Close application or minimize for later use

### Batch Processing

For processing multiple parts:

1. Keep application window open
2. Search first part number
3. Print required label
4. Clear the search field (click to select all, type new number)
5. Repeat for each part

<Note>
The application clears the placeholder text automatically when you click the search field, making it quick to enter a new search.
</Note>

## Troubleshooting Common Issues

### No Results Found

**Symptom**: Results table appears but is empty after searching.

**Solutions**:
- Verify the supplier part number is correct
- Check for extra spaces before or after the part number
- Confirm the part number exists in the Warehouse Support Dashboard database
- Try searching for a known valid part number to verify API connectivity

### Search Errors

**Symptom**: Error message appears: "Error en la solicitud: [error details]"

**Solutions**:
- Check your internet connection
- Verify the API endpoint is accessible (may require VPN)
- Wait a moment and try again - the API may be temporarily unavailable
- Contact your IT administrator if the error persists

### Application Not Responding

**Symptom**: Application freezes after clicking Search.

**Solutions**:
- Wait 10-15 seconds - the API may be slow to respond
- Check your network connection
- Close and restart the application
- If the issue persists, contact technical support
