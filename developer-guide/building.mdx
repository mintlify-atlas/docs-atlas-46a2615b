## Project Configuration

### Project File Structure

The project is configured in `Etiquetas.csproj`:

```xml Etiquetas.csproj
<Project Sdk="Microsoft.NET.Sdk">
  <PropertyGroup>
    <OutputType>WinExe</OutputType>
    <TargetFramework>net8.0-windows</TargetFramework>
    <Nullable>enable</Nullable>
    <UseWindowsForms>true</UseWindowsForms>
    <ImplicitUsings>enable</ImplicitUsings>
  </PropertyGroup>

  <ItemGroup>
    <PackageReference Include="System.Net.Http" Version="4.3.4" />
  </ItemGroup>
</Project>
```

### Configuration Properties

<ParamField path="OutputType" type="string">
  `WinExe` - Windows executable (no console window)
</ParamField>

<ParamField path="TargetFramework" type="string">
  `net8.0-windows` - .NET 8.0 with Windows-specific APIs
</ParamField>

<ParamField path="UseWindowsForms" type="bool">
  `true` - Enables Windows Forms support
</ParamField>

<ParamField path="ImplicitUsings" type="bool">
  `true` - Enables implicit global usings
</ParamField>

<ParamField path="Nullable" type="string">
  `enable` - Enables nullable reference types
</ParamField>

## Building the Application

### Debug Build

Build in Debug configuration with full debugging symbols:

```bash
dotnet build
```

Or with explicit configuration:

```bash
dotnet build --configuration Debug
```

<ParamField path="Output Location" type="path">
  `bin/Debug/net8.0-windows/`
</ParamField>

<ParamField path="Debug Symbols" type="bool">
  Included (`.pdb` files)
</ParamField>

<ParamField path="Optimizations" type="bool">
  Disabled
</ParamField>

### Release Build

Build optimized for production:

```bash
dotnet build --configuration Release
```

<ParamField path="Output Location" type="path">
  `bin/Release/net8.0-windows/`
</ParamField>

<ParamField path="Optimizations" type="bool">
  Enabled
</ParamField>

<ParamField path="Debug Symbols" type="bool">
  Optional (can be generated separately)
</ParamField>

### Clean Build

Remove all build artifacts:

```bash
dotnet clean
```

Clean and rebuild:

```bash
dotnet clean && dotnet build
```

## Publishing for Deployment

### Self-Contained Deployment

Create a standalone executable with bundled .NET runtime:

```bash
dotnet publish --configuration Release --runtime win-x64 --self-contained true
```

<ParamField path="Output Location" type="path">
  `bin/Release/net8.0-windows/win-x64/publish/`
</ParamField>

<ParamField path="Requires .NET Runtime" type="bool">
  No - Runtime is included
</ParamField>

<ParamField path="File Size" type="string">
  Larger (~70-100 MB with runtime)
</ParamField>

### Framework-Dependent Deployment

Create a deployment that requires .NET runtime to be installed:

```bash
dotnet publish --configuration Release --runtime win-x64 --self-contained false
```

<ParamField path="Output Location" type="path">
  `bin/Release/net8.0-windows/win-x64/publish/`
</ParamField>

<ParamField path="Requires .NET Runtime" type="bool">
  Yes - .NET 8.0 must be installed on target machine
</ParamField>

<ParamField path="File Size" type="string">
  Smaller (~2-5 MB without runtime)
</ParamField>

### Single-File Executable

Publish as a single executable file:

```bash
dotnet publish --configuration Release --runtime win-x64 --self-contained true --output ./publish -p:PublishSingleFile=true
```

<ParamField path="Output" type="string">
  Single `Etiquetas.exe` file
</ParamField>

<ParamField path="Deployment" type="string">
  Copy single file to target machine
</ParamField>

## Development Workflow

### Using Visual Studio

1. **Open Solution**: Double-click `Etiquetas.sln`
2. **Build**: Press `Ctrl+Shift+B` or Build → Build Solution
3. **Run**: Press `F5` (with debugging) or `Ctrl+F5` (without debugging)
4. **Debug**: Set breakpoints, press `F5`, use debugger tools

### Using Visual Studio Code

1. **Open Folder**: Open the `Etiquetas` folder
2. **Install C# Extension**: Install C# Dev Kit extension
3. **Build**: Terminal → Run Task → `build`
4. **Run**: Press `F5` (requires launch.json configuration)

### Using Command Line

```bash
# Navigate to project directory
cd Etiquetas

# Restore dependencies
dotnet restore

# Build
dotnet build

# Run
dotnet run
```

## Debugging

### Enable Detailed Build Output

```bash
dotnet build --verbosity detailed
```

### Debug Configuration

The application is built with debug symbols by default in Debug configuration.

**Debug Features**:
- Full stack traces with line numbers
- Variable inspection
- Breakpoint support
- Exception details

### Console Output (Development)

To see console output during development, temporarily change `OutputType`:

```xml Etiquetas.csproj
<OutputType>Exe</OutputType>  <!-- Shows console window -->
```

Change back to `WinExe` for production builds.

## Common Build Issues

### Issue: SDK Not Found

**Error**: "The current .NET SDK does not support targeting .NET 8.0"

**Solution**: Install .NET 8.0 SDK or later
```bash
winget install Microsoft.DotNet.SDK.8
```

### Issue: Windows Forms Not Available

**Error**: "Windows Forms is not supported on this platform"

**Solution**: Ensure you're building on Windows and the SDK is `net8.0-windows`

### Issue: Missing Resources

**Error**: "Could not find resource 'Properties.Resources.Daikin_logo'"

**Solution**: Ensure resource files are present in `Properties/Resources.resx`

### Issue: HTTP Request Failures

**Error**: "Error en la solicitud: Unable to connect"

**Solution**: Check network connectivity and API endpoint availability

## Continuous Integration

### GitHub Actions Example

```yaml .github/workflows/build.yml
name: Build

on: [push, pull_request]

jobs:
  build:
    runs-on: windows-latest
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Setup .NET
      uses: actions/setup-dotnet@v3
      with:
        dotnet-version: 8.0.x
    
    - name: Restore dependencies
      run: dotnet restore
    
    - name: Build
      run: dotnet build --configuration Release --no-restore
    
    - name: Publish
      run: dotnet publish --configuration Release --runtime win-x64 --self-contained true
```
