## Overview

Daikin Etiquetas integrates with the McQuay Support Dashboard API to retrieve part number information. The integration is implemented using `HttpClient` with async/await patterns and JSON deserialization.

## API Endpoint

The application connects to a single API endpoint:

<ParamField path="Base URL" type="string">
  `https://azuappsrvuat01v.mcquay.com/SupportDashboard`
</ParamField>

<ParamField path="Endpoint" type="string">
  `/Home/GetReportData`
</ParamField>

<ParamField path="Environment" type="string">
  UAT (User Acceptance Testing)
</ParamField>

## Request Format

### HTTP Method

**GET** request with query parameters

### Query Parameters

<ParamField path="id" type="integer" required>
  Report ID - Fixed value of `2167`
</ParamField>

<ParamField path="parameters" type="string" required>
  JSON-encoded string containing the supplier part number query
</ParamField>

### Request URL Construction

The API request is constructed in the `GetDataFromApi` method:

```csharp Form1.cs:124-144
private async Task GetDataFromApi()
{
    using (HttpClient client = new HttpClient())
    {
        string supplierPartNumber = txtSupplierP_N.Text.Trim();
        string url = $"https://azuappsrvuat01v.mcquay.com/SupportDashboard/Home/GetReportData?id=2167&parameters={{\"supplier_part_number\":\"{supplierPartNumber}\"}}";
        
        try
        {
            HttpResponseMessage response = await client.GetAsync(url);
            response.EnsureSuccessStatusCode();
            string responseBody = await response.Content.ReadAsStringAsync();
            
            ProcessApiResponse(responseBody);
        }
        catch (HttpRequestException ex)
        {
            MessageBox.Show($"Error en la solicitud: {ex.Message}", 
                          "Error", MessageBoxButtons.OK, MessageBoxIcon.Error);
        }
    }
}
```

### Example Request URL

For supplier part number `ABC-12345`:

```text
https://azuappsrvuat01v.mcquay.com/SupportDashboard/Home/GetReportData?id=2167&parameters={"supplier_part_number":"ABC-12345"}
```

The API returns a JSON array of objects:

```json

  },

    "Supplier_Part_number": "ABC-12345",
    "Daikin_Part_Number": "DKN-67891"
  }

```

### Response Fields

<ResponseField name="Supplier_Part_number" type="string" required>

</ResponseField>

<ResponseField name="Daikin_Part_Number" type="string" required>

</ResponseField>

<Note>

</Note>

## Response Processing

The `ProcessApiResponse` method deserializes and displays the results:

```csharp Form1.cs:146-167
private void ProcessApiResponse(string jsonResponse)
{
    try
    {
        daikinPartN_tbl.Rows.Clear();
        
        var results = JsonSerializer.Deserialize<List<Dictionary<string, object>>>(jsonResponse);
        
        foreach (var result in results)
        {
            daikinPartN_tbl.Rows.Add(
                "Print",
                result["Supplier_Part_number"],
                result["Daikin_Part_Number"]
            );
        }
    }
    catch (JsonException ex)
    {
        MessageBox.Show($"Error procesando la respuesta: {ex.Message}", 
                      "Error", MessageBoxButtons.OK, MessageBoxIcon.Error);
    }
}
```

### Deserialization Details

<ParamField path="Type" type="generic">
  `List<Dictionary<string, object>>`
</ParamField>

<ParamField path="Serializer" type="library">
  `System.Text.Json.JsonSerializer`
</ParamField>

The response is deserialized as a list of dictionaries to handle dynamic JSON structure without requiring a strongly-typed model class.

## HttpClient Usage

### Client Lifecycle

```csharp Form1.cs:126
using (HttpClient client = new HttpClient())
```

The `HttpClient` is instantiated within a `using` statement to ensure proper disposal after each request.

<Warning>
  **Best Practice Note**: For production applications, consider using a singleton or `IHttpClientFactory` to avoid socket exhaustion. The current implementation creates a new `HttpClient` for each request, which is acceptable for low-frequency operations but not optimal for high-volume scenarios.
</Warning>

### Request Flow

1. **Create HttpClient** - New instance per request
2. **Build URL** - Interpolate supplier part number into query string
3. **Send GET Request** - `await client.GetAsync(url)`
4. **Validate Response** - `response.EnsureSuccessStatusCode()`
5. **Read Content** - `await response.Content.ReadAsStringAsync()`
6. **Process Response** - Pass JSON string to `ProcessApiResponse()`
7. **Dispose Client** - Automatic via `using` statement

## Async/Await Pattern

### Async Event Handler

```csharp Form1.cs:95

```

The button click event handler is marked `async void` to allow awaiting the API call while keeping the UI responsive.

### Async Task Method

```csharp Form1.cs:124
private async Task GetDataFromApi()
```

The actual API call is in an `async Task` method, following best practices for async methods.

### Await Chain

```csharp
await client.GetAsync(url)                    // Wait for HTTP response
await response.Content.ReadAsStringAsync()    // Wait for body read
```

Both I/O operations are awaited to prevent blocking the UI thread.
